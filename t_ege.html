  • Используйте сортировку для упорядочивания результатов
  • Применяйте агрегатные функции (COUNT, SUM, AVG) при необходимости
  • Проверяйте корректность объединения таблиц через JOIN

Пример задания:

На основе таблиц базы данных получите список товаров:

Products (ID, Name, Price, Category)
Orders (OrderID, ProductID, Quantity)
Categories (CategoryID, CategoryName)

Необходимо:
1. Найти все товары дороже 1000 рублей
2. Сгруппировать по категориям
3. Показать только категории с более чем 5 товарами

Решение:

SELECT c.CategoryName, COUNT(p.ID) as ProductCount
FROM Categories c
JOIN Products p ON c.CategoryID = p.Category
WHERE p.Price > 1000
GROUP BY c.CategoryName
HAVING COUNT(p.ID) > 5
ORDER BY ProductCount DESC;

  • Следите за тем, чтобы рекурсия всегда приводила к базовому случаю
  • Используйте мемоизацию для оптимизации при повторяющихся вычислениях
  • Рассмотрите возможность замены рекурсии на итерацию при большой глубине
  • Внимательно отслеживайте граничные условия

Типичные ошибки:

  • Отсутствие базового случая
  • Бесконечная рекурсия
  • Неправильная обработка граничных случаев
  • Избыточные рекурсивные вызовы

Пример задания:

Напишите рекурсивный алгоритм для вычисления чисел Фибоначчи.

Решение:

def fib(n):
    # Базовый случай
    if n <= 1:
        return n
    # Рекурсивный шаг    
    return fib(n-1) + fib(n-2)

# Оптимизированная версия с мемоизацией
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

Анализ решения:

  • Базовый случай: n ≤ 1
  • Рекурсивный шаг: F(n) = F(n-1) + F(n-2)
  • Оптимизация: использование словаря memo для хранения промежуточных результатов

    def is_valid(x, y):
        return 0 <= x < n and 0 <= y < m and not field[x][y] == '#'
    
    def dfs(x, y):
        nonlocal coins
        if not is_valid(x, y) or visited[x][y]:
            return
            
        visited[x][y] = True
        if field[x][y] == 'C':
            coins += 1
            
        # Проверяем все возможные направления
        for dx, dy in [(0, 1), (1, 0), (0, -1), (-1, 0)]:
            new_x, new_y = x + dx, y + dy
            dfs(new_x, new_y)
    
    dfs(0, 0)
    return coins
